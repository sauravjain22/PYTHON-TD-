What is a literal in Python?

In Python, a literal is a fixed value written directly in the code.

👉 It is the actual data itself, not stored in a variable, not calculated.

Simple idea:

Literal = raw value

Basic Examples
10
3.14
"Hello"
True


All of these are literals because they are written directly.

Literal vs Variable (Very Important)
x = 10

10 → literal

x → variable (just a name)

x stores the literal 10

Another example:

name = "Saurabh"

"Saurabh" → string literal

name → variable

Types of Literals in Python

1️⃣ Numeric Literals
100      # integer literal
3.5      # float literal
2+3j     # complex literal

2️⃣ String Literals
"Hello"
'Python'
"""This is a multi-line string"""

All are string literals.

3️⃣ Boolean Literals

Python has only two:

True
False

⚠️ Capital T and F are mandatory.

4️⃣ Special Literal
None

Used to represent nothing / empty / no value.

5️⃣ Collection Literals

These create data structures directly.

[1, 2, 3]           # list literal
(10, 20)            # tuple literal
{"a": 1, "b": 2}    # dictionary literal
{1, 2, 3}           # set literal

Type Conversion and Type Casting in Python (In Depth)
1. What Does “Type” Mean in Python?

In Python, every value has a data type, which defines:

what kind of data it is, and

what operations can be performed on it.

Examples:

10 → int

10.5 → float

"10" → str

True → bool

Sometimes, we need to change one data type into another.
This process is called type conversion or type casting.

2. Why Type Conversion Is Necessary

Python is strict about data types.

Example:

a = input("Enter a number: ")
print(a + 5)


❌ This raises an error because:

input() returns a string

5 is an integer

To perform arithmetic, the string must be converted to a number.

3. Implicit Type Conversion (Automatic)
Definition

Implicit type conversion occurs when Python automatically converts one data type into another during an operation, without any instruction from the programmer.

Python only performs safe conversions.

Example 1: int + float
x = 10        # int
y = 3.5       # float

result = x + y
print(result)
print(type(result))


Output:

13.5
<class 'float'>


Explanation:

Python converts int → float

This prevents data loss and preserves precision

Example 2: Boolean Conversion
a = 5
b = True

print(a + b)

Output:

6

Explanation:

True is internally treated as 1

False is treated as 0

What Python Will NOT Do
x = 10
y = "5"

print(x + y)   # TypeError


Python does not implicitly convert strings to numbers because it is unsafe.

Type Promotion Order (Priority)
bool → int → float → complex


Python always promotes to the higher-precision type.

4. Explicit Type Conversion (Type Casting)
Definition

Explicit type conversion, also known as type casting, is when the programmer manually converts one data type into another using built-in functions.

5. Common Type Casting Functions
Function	Converts To
int()	    Integer
float()	    Floating-point number
str()	    String
bool()	    Boolean
complex()	Complex number

6. int() — Detailed Behavior
int(10.9)     # 10 (decimal part removed)
int(True)     # 1
int(False)    # 0


Invalid conversions:

int("10.5")   # ValueError
int("abc")    # ValueError


Valid:

int("10")

7. float() — Detailed Behavior
float(10)        # 10.0
float("10.5")    # 10.5
float(True)      # 1.0

8. str() — Practical Usage

Used mainly for:

Displaying output

String concatenation

age = 25
print("Age is " + str(age))

Without casting, this would raise an error.

9. bool() — Truth Value Testing (Very Important)
Values That Convert to False
bool(0)
bool(0.0)
bool("")
bool(None)

Values That Convert to True
bool(1)
bool(-5)
bool("Python")
bool(" ")


Rule:

Empty or zero values are False; everything else is True.

10. complex() — Basic Example
complex(5)        # (5+0j)
complex(2, 3)     # (2+3j)

11. Real-World Example (User Input)
a = input("Enter first number: ")
b = input("Enter second number: ")

total = int(a) + int(b)
print(total)


Explanation:

input() → string

int() → numeric conversion

Arithmetic becomes possible

12. Type Conversion vs Type Casting (Clear Comparison)
Aspect	        Type Conversion	    Type Casting
Performed by	    Python	        Programmer
Automatic	         Yes	              No
Safety	         Always safe	    Can cause errors
Example	           int → float	        str → int

13. Common Beginner Mistakes
Mistake 1: String Concatenation
"10" + "20"   # "1020"


Correct:

int("10") + int("20")   # 30

Mistake 2: Forgetting to Cast Input
age = input("Enter age: ")
print(age + 1)   # TypeError

Correct:

print(int(age) + 1)

14. Key Takeaways

input() always returns a string

Python performs implicit conversion only when safe

Use type casting when you need control

Errors during casting are common and normal


🔹 What are Operators in Python?

Operators are special symbols that tell Python to perform an action on values or variables.

👉 In simple words:
Operators = action
Operands = data

Example:

a = 10
b = 5
c = a + b

Here:

+ → operator

a and b → operands

1️⃣ Arithmetic Operators (Math Operations)

Used for calculations like you do in real life.

Operator	Meaning	Example
+	Addition	10 + 5 = 15
-	Subtraction	10 - 5 = 5
*	Multiplication	10 * 5 = 50
/	Division	10 / 5 = 2.0
%	Modulus (remainder)	10 % 3 = 1
//	Floor division	10 // 3 = 3
**	Power	2 ** 3 = 8

🔹 Real Example: Shopping Bill
price = 499
quantity = 3

total = price * quantity
print(total)

🔹 Real Example: Even or Odd
number = 17

if number % 2 == 0:
    print("Even")
else:
    print("Odd")

2️⃣ Assignment Operators (Store & Update Values)

Used to assign or update values.

Operator Meaning
=	Assign
+=	Add and assign also known as compound operator
-=	Subtract and assign
*=	Multiply and assign
/=	Divide and assign
🔹 Real Example: Bank Balance
balance = 1000
balance += 500   # salary credited
balance -= 200   # shopping

print(balance)


💡 Same as:

balance = balance + 500

But shorter and cleaner.

3️⃣ Comparison Operators (True / False Check)

Used to compare values.
Result is always True or False.

Operator	Meaning
==	Equal
!=	Not equal
>	Greater than
<	Less than
>=	Greater or equal
<=	Less or equal
🔹 Real Example: Exam Result
marks = 72

if marks >= 40:
    print("Pass")
else:
    print("Fail")

🔹 Real Example: Login Check
saved_password = "admin123"
entered_password = "admin123"

print(saved_password == entered_password)

4️⃣ Logical Operators (Decision Making)

Used when multiple conditions are involved.

Operator	Meaning
and	        Both must be True
or	        Any one True
not     	Reverse result
🔹 Real Example: Job Eligibility
age = 22
degree = True

if age >= 21 and degree:
    print("Eligible")
else:
    print("Not eligible")

🔹 Real Example: Weekend Check
day = "Sunday"

if day == "Saturday" or day == "Sunday":
    print("Holiday")

5️⃣ Identity Operators (Same Object or Not)

Check memory location, not value.

Operator	Meaning
is	        Same object
is not	    Different object
🔹 Example
a = [1, 2, 3]
b = a
c = [1, 2, 3]

print(a is b)   # True
print(a is c)   # False


💡 Use this mostly with None.

x = None

if x is None:
    print("No value")

6️⃣ Membership Operators (Present or Not)

Check if a value exists in a collection.

Operator	Meaning
in	        Exists
not in	    Does not exist

🔹 Real Example: Username Check
users = ["rahul", "saurav", "neha"]

if "saurav" in users:
    print("User found")

🔹 Real Example: Spam Word Filter
message = "win a free prize"

if "free" in message:
    print("Spam detected")

7️⃣ Bitwise Operators (Low-level, Advanced)

Used mostly in hardware, networking, optimizations.

    Operator	Meaning
        &	AND
        `	`
        ^	XOR
        ~	NOT
        <<	Left shift
        >>	Right shift
        🔹 Example
        a = 5   # 101
        b = 3   # 011

print(a & b)   # 1

🔹 What is Operator Precedence?

Precedence decides which operator runs first when multiple operators are in one expression.

👉 Just like BODMAS in maths.

Example:

result = 10 + 5 * 2
print(result)

❓ What do you expect?

10 + 5 = 15 → 15 * 2 = 30 ❌ (WRONG)

✅ Actual Output:
20


Because:

* has higher precedence than +

So Python does: 5 * 2 = 10

Then: 10 + 10 = 20

🔹 Common Operator Precedence (High → Low)
Priority	Operators
1️⃣	()
2️⃣	**
3️⃣	* / // %
4️⃣	+ -
5️⃣	< > <= >= == !=
6️⃣	not
7️⃣	and
8️⃣	or
9️⃣	= += -= *=

💡 Golden rule:
👉 Brackets always win

🔹 Example 1: Without Brackets vs With Brackets
print(100 - 20 * 3)


Output:

40


Because:

20 * 3 = 60
100 - 60 = 40


Now with brackets:

print((100 - 20) * 3)


Output:

240

🔹 What is Associativity?

Associativity decides direction of evaluation
👉 Left to Right or Right to Left
👉 When two operators have same precedence

🔹 Left-to-Right Associativity (Most Operators)
Example:
result = 20 - 5 - 3
print(result)


Step-by-step:

20 - 5 = 15
15 - 3 = 12


✔ Left → Right

🔹 Right-to-Left Associativity (**, assignment)
Example: Power Operator
print(2 ** 3 ** 2)

❌ Wrong thinking:
(2 ** 3) ** 2 = 64

✅ Actual evaluation:
2 ** (3 ** 2)
2 ** 9 = 512


✔ Because ** works Right → Left

🔹 Assignment Operator Associativity
a = b = c = 10
print(a, b, c)


✔ Evaluated as:

a = (b = (c = 10))


Output:

10 10 10

🔹 Mixing Precedence + Associativity (Real Example)
result = 10 + 20 * 3 ** 2
print(result)

Step-by-step:

1️⃣ ** → 3 ** 2 = 9
2️⃣ * → 20 * 9 = 180
3️⃣ + → 10 + 180 = 190

Output:

190

🔹 Logical Operators Precedence
print(True or False and False)

Evaluation:

and has higher precedence than or

So:

False and False → False
True or False → True


Output:

True

🔹 Real-Life Style Example (Eligibility System)
age = 22
has_id = True
is_student = False

if age >= 18 and has_id or is_student:
    print("Allowed")

How Python reads it:
(age >= 18 and has_id) or is_student


👉 This may give unexpected result.

✅ Correct & Safe Code:
if (age >= 18 and has_id) or is_student:
    print("Allowed")


💡 Always use brackets for clarity — even if you know precedence.

🔹 Quick Interview Tricks ⚠️
❓ Question:
print(5 + 3 > 6 and 2 ** 2 == 4)

Step-by-step:

1️⃣ + → 5 + 3 = 8
2️⃣ > → 8 > 6 = True
3️⃣ ** → 2 ** 2 = 4
4️⃣ == → 4 == 4 = True
5️⃣ and → True and True = True

✔ Output:

True

✅ Final Golden Rules (Remember Forever)

🟢 () beats everything
🟢 ** is right associative

Rule:

👉 Har saal interest new total pe lagega

Year 1:

10% of 1000 = ₹100

Total = ₹1100

Year 2:

10% of 1100 = ₹110

Total = ₹1210

👉 Final Amount = ₹1210

📌 Yaha interest badhta gaya

👉 Compound Interest
= har saal interest badhta hai
= interest interest pe bhi lagta hai

What is List Comprehension?

List comprehension is a short and smart way to create a list using:

a loop

optional condition

in one single line

👉 It replaces long for-loops.

Normal Way (Old Style)
numbers = [1, 2, 3, 4, 5]
squares = []

for n in numbers:
    squares.append(n * n)

print(squares)

List Comprehension Way (Modern Style)
numbers = [1, 2, 3, 4, 5]
squares = [n * n for n in numbers]

print(squares)


✨ Same output, less code, more readability.

Basic Syntax (Very Important)
new_list = [expression for item in iterable]


With condition:

new_list = [expression for item in iterable if condition]

Example 1: Even Numbers (Real Use)
Normal
numbers = [1, 2, 3, 4, 5, 6]
even = []

for n in numbers:
    if n % 2 == 0:
        even.append(n)

print(even)

List Comprehension
even = [n for n in numbers if n % 2 == 0]
print(even)

Example 2: Employee Salary Bonus (Corporate Example)

👉 Give 10% bonus to salaries above 30,000

Example 3: Convert Strings to Uppercase

upper_names = [names.upper() for name in names]
print(upper_names)


Example 4: Conditional Expression (If–Else inside)

print(result)


📌 Interview favorite pattern.

Example 5: Remove Empty Values (Very Real)

Used in data cleaning, Excel imports, APIs.

[[1, 0, 0],
 [0, 0, 0],

What is id() in Python?

id() is a built-in Python function that tells you the unique identity of an object in memory.

👉 Think of it like a house address for a variable in RAM.

id(object)


It returns an integer that stays unique for that object during its lifetime.

🔹 Simple Example
a = 10
print(id(a))


Now:

b = 10
print(id(b))


💡 Output will be same in many cases.

Why?
Because Python reuses memory for small integers (−5 to 256). This is called integer caching.

🔹 Example with Different Objects
a = 1000
b = 1000

print(id(a))
print(id(b))


📌 Output → Different IDs

Why?
Because large integers are stored separately in memory.

🔹 id() with Lists (Important Interview Point)
list1 = [1, 2, 3]
list2 = [1, 2, 3]

print(id(list1))
print(id(list2))


❌ IDs are different
✔ Because lists are mutable (changeable)

🔹 Same Reference, Same ID
list1 = [10, 20]
list2 = list1

print(id(list1))
print(id(list2))


✔ Same ID
✔ Both variables point to same memory location

🔹 id() with Strings

s1 = "hello"
s2 = "hello"

print(id(s1))
print(id(s2))


✔ Same ID
Because Python uses string interning to save memory.

But:

s1 = "hello world!"
s2 = "hello world!"

print(id(s1))
print(id(s2))


IDs may differ depending on optimization.

🔹 Real-Life Analogy 🧠
Python Term	Real Life
Object	    House
Variable	Person name
id()	    House address

Two people can point to the same house → same id.

🔹 id() vs == vs is (VERY IMPORTANT)
a = [1, 2]
b = [1, 2]

print(a == b)   # True (values same)
print(a is b)   # False (memory different)
print(id(a) == id(b))  # False


📌 == → checks value
📌 is → checks memory address
📌 id() → shows that address

🔹 Common Interview Questions

✔ CPython: Yes (practically)
✔ Officially: It’s just a unique identifier

❓ Can two objects have same id()?

❌ No (at the same time)

❓ Does id() change?

✔ If object is destroyed and memory reused → possible

🔹 When to Use id() in Real Code?

✔ Debugging
✔ Understanding reference vs copy
✔ Memory behavior
✔ Learning Python internals
❌ Not used in business logic

🔹 Quick Summary

Same object → same id

Mutable objects → different ids

Used with is for reference checks

Mostly for learning & debugging

🧠 Shallow Copy vs Deep Copy (Python)
👉 1. Shallow Copy

💡 Meaning:

A shallow copy creates a new object,
BUT inside objects (nested items) are still shared (same reference).

📌 Example:
import copy

original = [[1, 2], [3, 4]]
shallow = copy.copy(original)

shallow[0][0] = 100

print(original)  # [[100, 2], [3, 4]]
print(shallow)   # [[100, 2], [3, 4]]
⚠️ Why?
Outer list = copied ✅
Inner lists = SAME reference ❌

👉 So change inside → affects both

👉 2. Deep Copy

💡 Meaning:

A deep copy creates a completely independent copy
👉 everything is duplicated (no shared reference)

📌 Example:
import copy

original = [[1, 2], [3, 4]]
deep = copy.deepcopy(original)

deep[0][0] = 100

print(original)  # [[1, 2], [3, 4]]
print(deep)      # [[100, 2], [3, 4]]
✅ Why?
Outer list = copied
Inner list = ALSO copied

👉 So change doesn’t affect original
➡ copy() copies references, not nested objects

1️⃣ What is a Copy?

A copy means creating a new object from an existing one.

Example:

a = [1, 2, 3]
b = a


❌ This is NOT a copy
This is just another reference to the same object.

2️⃣ Shallow Copy
🔹 What is Shallow Copy?

A shallow copy creates a new outer object,
but inner (nested) objects are shared.

👉 Changes in nested data affect both copies.

🔹 Shallow Copy Example (IMPORTANT)
import copy

original = [1, 2, [10, 20]]
shallow = copy.copy(original)

shallow[0] = 100
shallow[2][0] = 999

print(original)
print(shallow)

Output:
[1, 2, [999, 20]]
[100, 2, [999, 20]]

🔍 Explanation:

shallow[0] = 100 → only shallow changed

shallow[2][0] = 999 → BOTH changed ❌

Because:
👉 Nested list is shared

🔹 How Shallow Copy Works (Memory idea)
original  ──► [1, 2, ──► [10, 20]]
shallow   ──► [1, 2, ──► same inner list]

3️⃣ Deep Copy
🔹 What is Deep Copy?

A deep copy creates:

New outer object

New inner (nested) objects

👉 Completely independent copy

🔹 Deep Copy Example
import copy

original = [1, 2, [10, 20]]
deep = copy.deepcopy(original)

deep[0] = 100
deep[2][0] = 999

print(original)
print(deep)

Output:
[1, 2, [10, 20]]
[100, 2, [999, 20]]


✔ No change in original
✔ Safe for nested structures

🔹 Deep Copy Memory Idea
original ──► [1, 2, ──► [10, 20]]
deep     ──► [1, 2, ──► [10, 20]]


(Completely separate)

4️⃣ Real-Life Example 🧠
👨‍💼 Employee Records

    "name": "Saurabh",
    "skills": ["Python", "SQL"]

❌ Shallow Copy Problem

✅ Deep Copy Solution

In Python, there are mainly 5 types of arguments:

1️⃣ Positional Arguments

Because values are assigned based on position.

2️⃣ Keyword Arguments

👉 Order does NOT matter.

    print("Age:", age)

✔ Here Python matches by parameter name, not position.

Answer: Improves readability and avoids positional mistakes.

👉 You provide default value in function definition.

    print("Age:", age)

Important Rule ⚠:
Default parameters must come after normal parameters.

❌ Wrong:

Why must default arguments come after non-default arguments?
Because Python assigns arguments from left to right.

👉 When you don’t know how many arguments user will pass.

Logging systems

Dynamic API inputs

Frameworks like Django

Flask

Django

FastAPI

Configuration systems

🔁 What is a Recursive Function?

A recursive function is a function that calls itself until a stopping condition (base case) is met.

Structure:
def function_name(parameters):
    if base_condition:
        return value
    else:
        return function_name(smaller_problem)

🧠 Two Important Parts (Very Important for Interviews)

Base Case
→ Stops infinite recursion
→ Without this → Stack Overflow Error

Recursive Case
→ Function calls itself with smaller input

Each subfolder can contain more subfolders.

This is recursive structure.

When a local variable has the same name as a global variable, the local variable overrides (shadows) the global variable inside the function.

x = 50   # global

def test():
    x = 100   # local
    print(x)

test()
print(x)


Output:

100
50


👉 The global variable remains unchanged.

5️⃣ How can you modify a global variable inside a function?

Answer:

We use the global keyword.

x = 10

def change():
    global x
    x = x + 5

change()
print(x)


Without global, Python treats it as a local variable.

6️⃣ Why are global variables considered bad practice?

Answer:

Because:

They make code tightly coupled

Hard to debug

Unsafe in multi-threading

Difficult to test

Can cause unexpected side effects

In real projects, we prefer passing parameters instead of using globals.

🔴 ADVANCED LEVEL
7️⃣ What is the LEGB Rule?

Answer:

Python searches variables in this order:

L → Local
E → Enclosing
G → Global
B → Built-in


Example:

x = "global"

def outer():
    x = "enclosing"
    
    def inner():
        print(x)

    inner()

outer()


Output:

enclosing


Because Python found it in Enclosing scope.

8️⃣ What is the difference between global and nonlocal?

Answer:

global	nonlocal
Refers to global scope	Refers to enclosing function
Used in normal functions	Used in nested functions

Example:

def outer():
    x = 10
    
    def inner():
        nonlocal x
        x = 20
    
    inner()
    print(x)

outer()


Output:

20

9️⃣ When does Python decide variable scope?

Answer:

Python decides variable scope at compile time (before execution).
If a variable is assigned inside a function, Python treats it as local unless specified global or nonlocal.

🔹 What is filter()?

filter() is a built-in Python function used to:

👉 Select elements from an iterable
👉 Based on a condition
👉 Using a function

🔹 Syntax

function → returns True or False

iterable → list, tuple, etc.

Returns → filter object (iterator)

🟢 Basic Example

numbers = [1, 2, 3, 4, 5, 6]

Output:

[2, 4, 6]

Explanation:

For each number

If condition True → keep it

If False → remove it

🧠 Important Concept

filter() does NOT modify original list.

It returns a filter object, so we convert it to list.

1️⃣ Without with Statement (Traditional Way)
file = open("data.txt", "r")
content = file.read()

What happens here?

open() → opens the file

read() → reads the content

close() → you manually close the file

⚠ Problem

If an error occurs before file.close():

file = open("data.txt", "r")
content = file.read()
print(10 / 0)   # Error happens here
file.close()    # This line will never execute

The file remains open.

This can cause:

Memory leaks

Resource exhaustion

Application crashes (in large systems)

2️⃣ With with Statement (Modern & Recommended Way)
with open("data.txt", "r") as file:
    content = file.read()
    print(content)
What happens here?

The file is opened.

The block inside with runs.

When the block finishes (even if an error happens), the file is automatically closed.

Even if this fails:

with open("data.txt", "r") as file:
    content = file.read()
    print(10 / 0)   # Error

The file will still be closed properly.

🔥 Why Does This Work?

The with statement uses something called a Context Manager.

It internally calls:

__enter__() → when the block starts

__exit__() → when the block ends

The file object in Python implements this protocol.

Without with	With with
You must manually close the file	File closes automatically
Risk of forgetting to close	No risk
Error can leave file open	File closes even after error
More error-prone	Cleaner and safer

Serialization means:

Converting a Python object into a format that can be stored or transmitted.

Usually into:

JSON string

File

Byte stream

In simple words:
👉 Python object → JSON/String/File

2️⃣ What is Deserialization?

Deserialization means:

Converting stored data (JSON/String/File) back into a Python object.

In simple words:
👉 JSON/String/File → Python object

🔥 Why Do We Need It?

Real corporate examples:

Sending API request (backend sends JSON)

Saving user data in file

Storing ML model

Caching data in Redis

Sending object over network

🧠 Most Common Way: JSON Module

Python provides a built-in module:

import json
🟢 Example 1: Serialization (Python → JSON String)
import json

student = {
    "name": "Saurabh",
    "age": 22,
    "course": "MSc IT"
}

json_data = json.dumps(student)

print(json_data)
print(type(json_data))
Output:
{"name": "Saurabh", "age": 23, "course": "MSc IT"}
<class 'str'>
Important:

dumps() → converts dictionary → JSON string

Output type becomes str

🟢 Example 2: Deserialization (JSON String → Python)
import json

json_data = '{"name": "Saurabh", "age": 23, "course": "MSc IT"}'

python_object = json.loads(json_data)

print(python_object)
print(type(python_object))
Output:
{'name': 'Saurabh', 'age': 23, 'course': 'MSc IT'}
<class 'dict'>
Important:

loads() → converts JSON string → dictionary

Now the Important Part
Deep Explanation — Email Validation Regex

Pattern:

^[\w\.-]+@[\w\.-]+\.\w+$

Let’s break it step by step.

Step 1
^

Means

Start of the string

So email must start from here.

Step 2
[\w\.-]+

Inside brackets means allowed characters.

Allowed characters:

\w  → letters numbers underscore
\.  → dot
-   → dash

Examples allowed

john
john123
john.doe
john-doe

+

Means

one or more characters

So:

john
john.doe
user123

All valid.

Step 3
@

Literal character.

Every email must contain @

Example

john@gmail.com
Step 4
[\w\.-]+

Same logic again.

This represents domain name

Examples

gmail
yahoo
outlook
company-name
Step 5
\.

Dot before domain extension.

Example

gmail.com

We escape dot with \

Because normally:

. = any character

So we escape it to mean real dot.

Step 6
\w+

Domain extension.

Examples

com
org
net
edu
Step 7
$

End of string.

Ensures no extra characters after email.

Complete Example
import re

email = "john.doe@gmail.com"

pattern = r"^[\w\.-]+@[\w\.-]+\.\w+$"

if re.match(pattern, email):
    print("Valid Email")
else:
    print("Invalid Email")

1️⃣ What Are Quantifiers?

A quantifier tells regex how many times a character or pattern can repeat.

Without quantifier:

Pattern

a

Matches only

a

But with quantifier:

a+

Matches

a
aa
aaa
aaaa

So quantifiers control repetition.

2️⃣ Main Quantifiers in Regex
Quantifier	Meaning
*	0 or more
+	1 or more
?	0 or 1
{n}	exactly n times
{n,}	n or more times
{n,m}	between n and m
3️⃣ * (Zero or More)

Pattern:

ab*

Meaning:

a followed by zero or more b

Matches:

a
ab
abb
abbb

Python example:

import re

text = "a ab abb abbb"

print(re.findall(r"ab*", text))

Output

['a', 'ab', 'abb', 'abbb']
Use Case

Optional repeating characters.

Example log data:

error
errorr
errorrr

Pattern:

error*
4️⃣ + (One or More)

Pattern:

ab+

Meaning:

a followed by one or more b

Matches:

ab
abb
abbb

But NOT:

a

Python example:

import re

text = "a ab abb abbb"

print(re.findall(r"ab+", text))

Output

['ab', 'abb', 'abbb']
Use Case

Extracting numbers.

Pattern:

\d+

Example:

text = "speed 120 min 20"

print(re.findall(r"\d+", text))

Output

['120', '20']

Why?

\d = digit

So it captures full numbers.

5️⃣ ? (Zero or One)

Pattern:

colou?r

Matches:

color
colour

Python example:

9️⃣ Greedy vs Non-Greedy Quantifiers

By default quantifiers are greedy.

Greedy means:

Match as much as possible.

Example:

text = "<h1>Hello</h1><h1>World</h1>"

Pattern:

<h1>.*</h1>

Output

<h1>Hello</h1><h1>World</h1>

It captured everything.

Non-greedy:

.*?

Example:

re.findall(r"<h1>.*?</h1>", text)

Output

['<h1>Hello</h1>', '<h1>World</h1>']
Use Case

Parsing HTML / logs / XML.

1️⃣ What is finditer()?

finditer() finds all matches in a string, but instead of returning a list like findall(), it returns an iterator of match objects.

Simple idea:

findall() → gives only matched text

finditer() → gives full match information

That information includes:

matched text

start position

end position

groups

2️⃣ Basic Syntax
re.finditer(pattern, string)

Example:

import re

text = "Speed 120 and 95"

matches = re.finditer(r"\d+", text)

for match in matches:
    print(match)

Output

<re.Match object; span=(6, 9), match='120'>
<re.Match object; span=(14, 16), match='95'>

You see it returns Match objects, not strings.

example of finditer 
import re
pattern = r"[A-Z][a-z]+[-][0-9]{10}"
text = "python is a programming language and Saurav-9403338964,Tarun-8275461815,Mummy-9892505651"
result = re.finditer(pattern,text)
for r in result:
    print(r)

output:
<re.Match object; span=(37, 54), match='Saurav-9403338964'>
<re.Match object; span=(55, 71), match='Tarun-8275461815'>
<re.Match object; span=(72, 88), match='Mummy-9892505651'>

>1️⃣ What is re.sub()?

re.sub() means substitute.

It replaces all matches of a regex pattern with something else.

Simple idea:

Find pattern → Replace it.

Example idea:

"My OTP is 4598"

Replace digits with *

"My OTP is ****"
2️⃣ Syntax

Parameters:

Parameter	Meaning

replacement	what to replace it with
string	original text

import re

>1️⃣ What are Regex Flags?

Flags modify how the regex engine behaves.

Normally regex is case-sensitive.

Example:

Pattern

car

Text

Car

Result → ❌ No match (because C ≠ c)

But with IGNORECASE, case doesn't matter.

car = Car = CAR = cAr
2️⃣ IGNORECASE Flag

In Python it is written as:

re.IGNORECASE

Short form:

re.I

Meaning:

Match text without caring about uppercase or lowercase.

3️⃣ Syntax

General syntax:

re.search(pattern, string, flags)

Example:

import re

text = "Car bike truck"

result = re.search("car", text, re.IGNORECASE)

print(result)

Output → Match found.

Without flag → ❌ No match.

1️⃣ What is re.compile()?

re.compile() compiles a regex pattern into a regex object.

Simple idea:

Instead of writing the pattern again and again, we compile it once and reuse it.

Basic syntax:

import re

pattern = re.compile("car")

Now pattern is a regex object.

You can use it like this:

pattern.search(text)
pattern.findall(text)
pattern.sub("replacement", text)
2️⃣ Basic Example

Without compile():

import re

text = "car bike car bus"

print(re.findall("car", text))

Output

['car', 'car']

With compile():

import re

pattern = re.compile("car")

text = "car bike car bus"

print(pattern.findall(text))

Output

['car', 'car']

Result is the same.

So the question becomes:

👉 Why use compile?

3️⃣ Main Reason — Performance

Every time you run:

re.findall("car", text)

Python must:

1️⃣ Read the pattern
2️⃣ Parse it
3️⃣ Convert it into regex instructions
4️⃣ Execute it

If this runs thousands of times, that compilation repeats.

But with compile():

pattern = re.compile("car")

Python compiles once, then reuses it.

So repeated operations become faster.

example:
import re
pattern = r"\d+"
compiled_pattern = re.compile(pattern)
text = "python is a programming language and Saurav-9403338964,Tarun-8275461815,Mummy-9892505651"
result = re.findall(compiled_pattern,text)
print(result)

3️⃣ Why Use finditer()

Your choice of finditer() is actually good.

re.finditer(pattern, data)

returns match objects.

From each match you can get:

match.group()   → matched text
match.start()   → start position
match.end()     → end position

Example:

    print(match.group(), match.start())

Example output:

Carol_123@example.com 38

This is useful in log parsing or text analysis.
